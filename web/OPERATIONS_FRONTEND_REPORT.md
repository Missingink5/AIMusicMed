# AIMusicMed 最终运营版前端报告

## 本次实现

- 用户素材中心：私人音色、私人音乐库、素材额度和站内通知。
- 音色克隆：录音试听、权利声明、上传后自动创建、三个音色槽位和 30 天三次克隆额度展示。
- 私人曲库：七类主要情绪、多个标签、本地波形试听、裁剪区间、淡入淡出，以及“自动/较轻/标准/较强”四档响度。
- 授权记录：音色和音乐分别提供清晰确认文案；删除音色明确同时删除原始录音。
- 管理员运营台：用户与额度、音色、公共曲库、任务、作品、匿名统计、备份恢复、系统状态和审计日志。
- 管理员敏感操作：创建备份和恢复备份需要管理员邮箱六位验证码，并使用短期动作令牌。
- 新增文案使用 `app/lib/copy.zh-CN.ts` 字典，便于后续增加英文。

## 前端 API 契约

当前生产后端尚未实现的接口已集中定义在 `app/lib/api.ts`，包括：

- `/assets/quota`
- `/voices`、`/voices/clone`、`/voices/{id}`
- `/music-library`、`/music-library/tracks`、`/music-library/tracks/{id}`
- `/notifications`
- `/admin/voices`、`/admin/music-library`、`/admin/jobs`、`/admin/works`
- `/admin/stats/anonymous`、`/admin/backups`、`/admin/system/status`、`/admin/audit-log`
- `/admin/sensitive-actions/code/request` 与 `/verify`

前端对尚未启用的运营接口使用空状态或友好错误，不伪造成功结果。

## 安全与隐私边界

- 上传文件使用 `FormData`，不写入浏览器持久存储。
- 波形只在浏览器本地生成，用于试听和裁剪，不推断音乐情绪。
- 管理员统计界面不展示用户聊天正文。
- 备份与恢复按钮必须经过管理员邮箱二次验证码。
- 本阶段未修改后端、数据库、部署文件，也未进行线上部署。
